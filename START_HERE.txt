Problem: "Connection refused" or "Can't reach page"
Solution: Make sure the Terminal is still running the server

Problem: "file://" appears in browser address
Solution: You opened the HTML file directly - DON'T DO THIS!
         Start the server first, then go to http://localhost:8080

Problem: "HTTP 403 Error"
Solution: Port conflict! Port 8080 should work on Mac
         (Port 5000 is used by AirPlay on macOS)

Problem: Server won't start
Solution: Make sure port 8080 isn't being used by another app
         Check with: lsof -i :8080
